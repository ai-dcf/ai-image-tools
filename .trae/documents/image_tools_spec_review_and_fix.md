# 图片工具箱需求设计对齐与修复计划

## 摘要 (Summary)
本次计划旨在对当前的图片工具箱项目进行全盘检查，比对 `2026-04-10-image-tools-design.md` 设计规范（Spec），找出目前实现中遗漏或未完全满足需求的功能点，并制定相应的修复和补全计划。目标是使所有工具完全契合“场景预设优先，通用配置兜底”的核心理念及详尽的交互细节。

## 当前状态分析 (Current State Analysis)
目前项目已成功搭建 Next.js 14 框架，完成了 Zustand 状态管理及共享的 `ToolLayout` 布局，五大核心工具（压缩、裁剪、边框、文字、拼图）的**基础功能**均已跑通。

但经过详细比对，发现以下需求规范**未被满足或存在缺失**：
1. **图片压缩 (/tools/compress)**: 缺失“原图/压缩图”对比滑块、预估加载时间、自定义宽高尺寸缩放。
2. **图片裁剪 (/tools/crop)**: 缺失“朋友圈/头像(1:1)”预设、精确宽高(px)输入、水平/垂直翻转功能。
3. **添加边框 (/tools/border)**: 极简预设未严格实现“1px描边”，缺失统一边框宽度的调节。
4. **添加文字 (/tools/text)**: 缺失字体选择、字间距、行高、阴影、背景色块等样式调节；四大预设（如带双引号的金句、右下角水印）未实现特定的排版与位置逻辑。
5. **图片拼图 (/tools/collage)**: 缺失“拖拽图片交换位置”功能、上传自定义背景图功能、填充模式（Fill/Fit）切换；且点击预设模板时未能联动更新间距、圆角等具体参数；对比测评模板缺少中间的 "VS" 图标。

## 拟定变更 (Proposed Changes)

### 任务 1：完善图片压缩工具 (Compress)
- **文件**: `src/components/tools/compress/CompressorCore.tsx`
- **变更内容**:
  - **移除 AVIF 支持**: 基于用户确认，因为 `browser-image-compression` 不支持，从设计和 UI 中移除 AVIF 格式选项。
  - **实现对比滑块**: 使用原生 CSS `clip-path` 与 `<input type="range">` 组合，在预览区实现原图与压缩图叠加的拖拽对比效果。
  - **新增尺寸缩放**: 在“通用配置”中增加宽度和高度的数字输入框（支持锁定长宽比），在压缩前使用 Canvas 或直接通过 `browser-image-compression` 的 `maxWidthOrHeight` 选项进行调整。
  - **新增加载时间预估**: 按照一般网速（如 3G/4G/5G 平均速率）基于压缩后文件大小（Bytes）计算并展示预估加载时间。

### 任务 2：完善图片裁剪工具 (Crop)
- **文件**: `src/components/tools/crop/CropCore.tsx`
- **变更内容**:
  - **新增预设**: 补充“朋友圈/头像” (1:1) 比例。
  - **精确数值输入**: 在控制面板添加宽度和高度输入框，当用户输入时，更新 `aspect` 并在导出时约束最终尺寸。
  - **翻转功能**: 引入水平翻转（`scaleX`）和垂直翻转（`scaleY`）的状态与按钮，并同步修改 `getCroppedImg` 导出逻辑以支持镜像翻转。

### 任务 3：完善边框工具 (Border)
- **文件**: `src/components/tools/border/BorderCore.tsx`, `BorderCanvas.tsx`
- **变更内容**:
  - **极简预设优化**: 修正“极简细线”预设，将其调整为 `padding: 0` 且外部加一层 `1px` 描边（可通过 `Konva.Rect` 实现边框线）。
  - **统一调节**: 添加“统一边距”锁定/解锁按钮，开启时四个方向的滑块值保持同步。

### 任务 4：完善文字工具 (Text)
- **文件**: `src/components/tools/text/TextCore.tsx`, `TextCanvas.tsx`
- **变更内容**:
  - **扩充配置项**: UI 中新增字体（Font Family 下拉框）、字间距（Letter Spacing）、行高（Line Height）、阴影（模糊度与颜色）、背景色块配置。并同步更新到 `TextItem` 接口及 `TextCanvas` 的 `<Text>` 属性中（背景色块可利用 `Label` 和 `Tag` 组合实现）。
  - **强化预设逻辑**: 
    - 金句：自动添加双引号装饰，使用衬线字体。
    - 视频标题：应用醒目底色块。
    - 水印：设置透明度，并在画布初始化时将其位置自动计算至右下角。
    - 标签：右上角，圆角背景。

### 任务 5：完善拼图工具 (Collage)
- **文件**: `package.json`, `src/components/tools/collage/CollageCore.tsx`, `CollageCanvas.tsx`
- **变更内容**:
  - **引入拖拽库**: 运行 `npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities`，在左侧“已上传图片”列表和中间预览区实现拖拽排序（交换图片数组顺序）。
  - **参数联动**: 修改预设按钮点击事件，点击时不仅切换 `preset`，还要重置 `settings` 的 `gap`、`borderRadius` 和 `backgroundColor` 以匹配该预设的风格。
  - **上传背景图**: 在右侧配置区增加“上传背景图”按钮，使用户能用图片替换纯色背景。
  - **填充模式**: 增加 `Object fit (Fill/Fit)` 下拉框，动态改变 `img` 标签的 CSS `object-fit` 属性。
  - **对比图完善**: 在 `renderCompare` 视图中，使用绝对定位在两图交界处叠加一个 "VS" 图标（SVG）。

## 假设与决策 (Assumptions & Decisions)
- **AVIF 格式**: 基于用户确认，由于核心压缩库不支持且增加额外 WASM 负担过重，直接废弃对 AVIF 格式的支持。
- **拖拽库选型**: 基于用户确认，使用成熟的 `@dnd-kit/core` 实现拼图工具中的图片拖拽交换位置，以确保最佳的交互流畅度和可维护性。

## 验证步骤 (Verification Steps)
1. **压缩工具**: 上传图片后，拖动图片中间的滑块，检查是否能清楚对比原图与压缩图的噪点/清晰度；检查能否输入具体的宽/高值。
2. **裁剪工具**: 点击 1:1 预设，检查裁剪框是否变为正方形；点击水平/垂直翻转并导出，检查下载的图片是否正确镜像。
3. **文字工具**: 点击“金句”预设，检查文本两侧是否出现引号且字体带衬线；修改背景色块并检查画布是否实时生效。
4. **拼图工具**: 拖拽左侧的已上传图片，检查中间画布中的排版是否跟随改变；点击“对比测评”预设，检查是否有 VS 图标且配置面板参数自动更新。

# UI 重设计划：高级极简与克制风格 (Linear-Style Restraint)

## 摘要 (Summary)
本计划旨在运用 `frontend-design` 和 `frontend-skill` 技能的指导原则，对“图片工具箱”进行彻底的 UI 重塑。我们将摒弃常见的 Tailwind 卡片网格和同质化的 AI 生成设计感，转而采用 **“高级极简 / Linear 风格的克制”** 视觉方向。这意味着更平静的界面层级、极其严谨的留白、强有力的排版排布、最小化的装饰性边框（Chrome），以及高级而克制的交互动效。

## 当前状态分析 (Current State Analysis)
- **首页 (`src/app/page.tsx`)**：目前使用了一个标准的 Hero 区域，下方跟着一个常规的 3 列卡片网格，配以基础的 Lucide 图标和彩色背景。缺乏独特的品牌感知和视觉锚点。
- **布局 (`Header.tsx` & `ToolLayout.tsx`)**：顶部导航是一个带底边框的常规吸顶栏。工具布局拥有明显的边框和标准背景色，缺乏专业级应用的质感。
- **样式**：使用了标准的 Tailwind 默认样式（Zinc 色系，默认无衬线字体）。没有任何高级的动效或交互设计。

## 拟定变更 (Proposed Changes)

### 1. 全局样式与字体 (`src/app/globals.css` & `src/app/layout.tsx`)
- **视觉风格**：“高级极简 (Refined Minimalist)”。使用极深的对比度（纯白/纯黑背景），仅在绝对必要时使用极细的边框，并挑选单一且强有力的强调色（例如锐利的电光蓝或极致的高对比度黑白）。
- **字体排版**：引入极具辨识度的字体（例如优美的展示型衬线体配合干净的无衬线正文，或者一套骨架极佳的现代几何无衬线体），通过严格的字间距和行高规则来确立层级。
- **动效**：引入 `framer-motion` 以实现高级的页面过渡、顺滑的模块显现以及精致的微交互。

### 2. 首页重塑 (`src/app/page.tsx`)
- **Hero 区域**: 
  - 采用全屏无边距 (Full-bleed) 的海报级构图。
  - 使用巨大且自信的排版来传递品牌承诺。
  - 引入一个强有力的视觉锚点（例如微妙的抽象交互元素，或工具使用场景的高级质感图形），而非通用的说明文字。
- **导航/工具展示**:
  - 移除同质化的卡片网格。
  - 替换为极简的交互式列表，或结构极其严谨的非对称 Bento 布局。
  - 悬浮状态（Hover）应使用 `framer-motion` 来展现描述或微妙的背景切换，避免使用笨重的块阴影。
- **文案**：无情地精简文案，使其更具产品属性且掷地有声。移除所有填充词。

### 3. 应用框架与导航 (`src/components/layout/Header.tsx`)
- **顶部导航**：将其改造为极简的悬浮药丸式导航，或完全无边距且与 Hero 区域无缝融合的沉浸式导航。
- **层级关系**：使品牌名称成为最响亮的文本，导航链接在未悬浮时保持内敛。

### 4. 工具工作区 (`src/components/layout/ToolLayout.tsx`)
- **工作区 UI**：全面应用“Linear 风格的克制”。
  - 移除粗重的边框和不必要的阴影。
  - 使用极微妙的背景色阶（例如 `#fafafa` 到 `#ffffff`）来区分画布区和控制面板区。
  - 确保配置面板（右侧）信息密集但高度可读，依靠严格的对齐和字体层级来区分模块，而非使用一个个的包围盒。

## 假设与决策 (Assumptions & Decisions)
- **依赖库**：我们将安装 `framer-motion` 来处理入场动画、共享布局过渡和悬浮特效。
- **设计系统**：我们将继续使用 `shadcn/ui` 和 Tailwind，但会覆写默认组件（如按钮、卡片、输入框）的样式，使其更扁平、更锐利（减少圆角或使用完美的胶囊形），从而提升质感。
- **无功能变更**：这纯粹是一次前端 UI/UX 重塑。底层的图片处理逻辑（Canvas、Zustand、browser-image-compression）将保持不变。

## 验证步骤 (Verification Steps)
1. 运行开发服务器，验证首页 Hero 区域是否以全屏无边距方式渲染，并具备全新的排版体系。
2. 鼠标悬停在工具链接/列表上，确保 `framer-motion` 的动画丝滑且性能优异。
3. 打开一个工具页面（例如 `/tools/compress`），确保 `ToolLayout` 的感觉更像是一个专业的桌面端应用（极简的界面装饰，清晰的层级），而不是一个普通的网页。
4. 确保移动端响应式得到维持，将首个移动端视口视为一张具有连贯性的海报。
